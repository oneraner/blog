## 前言

[舊的部落格](https://oneraner.blogspot.com/)，由於是用文字編輯器的方式編輯新文章，因此程式碼的撰寫不太容易，
整體樣式風格太像網際網路剛開始發展的樣子，因此有了想換部落格的想法。

## 為什麼不是從 0 開始製作的 Blog 而是找現成的

因為省時間，在沒有工作經驗的時候的確都從 0 開始花數個月打造 project，為了證明自己的能力，但過了幾年，根據不要重複造輪子的原則，
如果有現成的為什麼不用？檢視功能跟修改的時候也跟維護公司舊專案一樣，必須看懂別人提供的模板功能自己再去修改它，這個網站不需要花大量時間
在一些不是專業的事情上，例如找授權圖片、UI 設計，或是每個 module 都要自己重刻。

## 為什麼是 Tailwind Nextjs Starter Blog

我希望可以找到一個簡明的設計，前端框架要是 React，如果只有 UI 的話我就會用 Next.js 自己重寫，CSS 最好是 TailwindCSS，
因為 CSS 命名規範可能會有落差，切分方式萬一還要找就很麻煩，後來找到了這個完全符合需求，還送了很多功能的 Blog。

## 目前的待辦

1. 優化寫一篇新文章的流程
2. 從舊的部落格完全遷移到這裡
